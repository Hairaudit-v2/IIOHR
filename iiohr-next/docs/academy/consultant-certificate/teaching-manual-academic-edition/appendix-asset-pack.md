# Appendix A — Supplementary HTML tools  
## Consultant Stream · Master Academic Edition

**Scope:** Supplementary learner-facing HTML pages (print-friendly tools and references), versioned with the Consultant Stream programme in the LMS.  
**Edition:** 1.1 · March 2025  

* * *

### Document role

This appendix is **academic master** metadata for **Appendix A**: filenames, purposes, and deployment notes. It aligns to the live **resource list** in the LMS and is distinct from legacy Word manuals. It supports **DOCX/PDF** export as part of the bound manual set.

* * *

## 1. Purpose

The HTML pack provides **print-friendly** and **on-screen** supplementary material: role boundaries, consultation scaffolding, screening prompts, handover templates, pre- and post-operative coordination aids, and patient education talking points. **None** replaces **informed consent**, clinical judgement, or local governance.

## 3. LMS linkage

Resources are registered for the Consultant Stream **programme** so lessons can surface **Download** or **Open (print to PDF)** actions per file type. Filename stability matters for deep links and curriculum audits.

* * *

## 4. Pedagogical notes

- Treat HTML pages as **adjunct** readings and workplace aids, not assessed clinical instruments unless a **Module** explicitly assigns them.  
- Faculty should remind learners that **local consent forms, policies, and employment contracts** prevail.  
- Checklists and templates **do not** constitute surgeon or anaesthesia sign-off; doubtful items require documentation and escalation.

* * *

## 5. Deployment and change control

- Version HTML alongside **Module** releases when content changes materially.  
- Prefer additive URLs; avoid silent overwrites that break bookmarks in published syllabi.  
- Accessibility: maintain heading hierarchy and contrast when editing HTML.
